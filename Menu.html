DDOS 공격

웹

여러 대의 PC를 이용해 타깃 서버에 방대한 양의 데이터를 퍼부어 서비스를 마비시키는 공격입니다.

자세히 보기 ▾

DDoS(분산 서비스 거부) 공격은 여러 대의 컴퓨터를 동원해 특정 서버나 웹사이트에 비정상적으로 많은 트래픽을 한꺼번에 보내서, 시스템을 마비시키고 일반 사용자들이 이용하지 못하게 만드는 공격입니다. 쉽게 말해, 좁은 입구에 수많은 사람을 몰아넣어 아무도 출입하지 못하게 가로막는 것과 비슷합니다.

API injection

웹

API 입력값에 악성 코드를 주입해 서버가 이를 실행 명령으로 처리하도록 유도함으로써 데이터를 탈취하거나 시스템을 제어하는 공격입니다.

자세히 보기 ▾

공격자가 API 엔드포인트를 통해 악의적인 데이터를 주입하여 서버 측의 로직을 조작하거나, 데이터베이스를 탈취하고, 시스템 명령을 실행하는 보안 취약점을 말합니다.유형으론 SQL injection,NoSQL 인젝션,명령어 인젝션,등이 있다.

XSS

웹

웹사이트에 함정(악성 코드)을 설치해 두고, 방문한 사용자의 개인정보를 낚아채는 공격입니다.

자세히 보기 ▾

악의적인 스크립트를 몰래 올려두어, 그 글을 클릭한 다른 사용자의 브라우저에서 해당 코드가 실행되게 만드는 공격입니다.이 공격을 통해 사용자의 로그인 세션 정보를 탈취하거나, 가짜 화면을 띄워 개인정보를 직접 입력하도록 유도하는 등 방문자를 직접적인 타깃으로 삼는 것이 특징입니다.

Broken Access Control

애플리케이션

액세스 제어란 사용자가 권한을 벗어나 행동할 수 없도록 정책을 시행하는 것입니다.

자세히 보기 ▾

액세스 제어가 취약하면 사용자는 주어진 권한을 벗어나 모든 데이터를 무단으로 열람∙수정 혹은 삭제할 수 있습니다. 대응 방안으로는 모든 리소스에 대한 접근 제어를 철저히 하고, 사용자에게 최소한의 권한만을 부여해야 합니다.

Insecure Designnnnnnnnnn

애플리케이션

안전하지 않은 설계란 누락되거나 비효율적인 제어 설계로 표현되는 다양한 취약점을 말합니다.

자세히 보기 ▾

안전하지않게설계된애플리케이션은개발완료후코드를수정하여도보안결함을완벽히방어하는데한계가있습니다. 따라서 설계단계에서부터보안을적절히고려해야 합니다.

Vulnerable and Outdated Components

애플리케이션

취약하고 오래된 요소는 지원이 종료되었거나 오래된 버전을 사용할 때 발생합니다.

자세히 보기 ▾

이를 방지하기 위해서는 불필요한 기능, 구성요소, 파일 등을 삭제할 필요가 있습니다. 또한 지속적인 모니터링으로 취약한 소프트웨어가 사용되고 있는지 확인해야합니다.

IP Header & ACL

네트워크

수업 시간에 배운 IP 헤더의 소스(출발지)와 데스티네이션(목적지) 주소를 분석하여, 인가되지 않은 외부 IP의 접근을 차단하는 기술입니다.

자세히 보기 ▾

네트워크의 가장 기초적이면서 강력한 보안 첫걸음입니다.

Secure Connection

네트워크

TCP의 3-Way Handshake 과정을 악용하는 공격(예: SYN Flood)으로부터 서버를 보호하는 기술입니다.

자세히 보기 ▾

수업 때 배운 신뢰성 있는 연결 방식을 보안 관점에서 재해석하여 안전한 데이터 전송을 보장합니다.

Secure Socket Layer

네트워크

네트워크 프로그래밍에서 배운 소켓(Socket)을 통해 데이터를 주고받을 때, 평문이 아닌 암호화된 상태로 전송하는 보안 기술입니다.

자세히 보기 ▾

중간에 데이터를 가로채도 내용을 읽을 수 없도록 만드는 현대 보안의 필수 요소입니다.
